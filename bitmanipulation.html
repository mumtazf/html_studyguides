1. Bits, bytes, and binary

A bit is a single 1 or 0. Computers store everything — numbers, letters, images, this webpage — as long sequences of bits.

A group of 8 bits is called a byte. One byte can represent 28 = 256 different values. That's enough for numbers 0–255 or for a single character of text.

Here's the number 13 stored in 8 bits:

70 60 50 40 31 21 10 01

Read right to left. Each position has a value: 1, 2, 4, 8, 16, 32, 64, 128 (each is double the last — powers of two). Add up the positions with a 1 in them:

8 + 4 + 1 = 13

Why powers of two? Binary is base-2 for the same reason decimal is base-10. In decimal, the digit positions are 1, 10, 100, 1000. In binary they're 1, 2, 4, 8, 16. Same idea, different base.

2. Reading binary and hex

Programmers write numbers three ways:

FormatPrefixExampleSame as
Decimal—1313
Binary0b0b110113
Hex0x0x0D13

Hex is compact binary. Each hex digit maps to exactly 4 bits, so 0xFF is 8 bits all on = 255, and 0x80 is just the top bit of a byte. When you see hex in code, mentally chunk it as groups of 4 bits. 

// all four of these are the same number: 173
const a = 173;
const b = 0b10101101;
const c = 0xAD;
const d = 0o255;   // octal, rarely used

3. The six bitwise operators

These operators work on numbers one bit at a time, in parallel across all positions. In most languages the symbols are the same as below (JavaScript, C, C++, Java, Python, Rust, Go all agree).

AND  &

The output bit is 1 only if both input bits are 1.

aba & b
000
010
100
111

Use it to mask off (keep) certain bits and zero the rest.

OR  |

The output bit is 1 if either input bit is 1.

aba | b
000
011
101
111

Use it to turn bits on.

XOR  ^

Exclusive OR: 1 if the bits differ.

aba ^ b
000
011
101
110

Use it to toggle bits, or to detect differences.

NOT  ~

Unary. Flips every bit: 0 becomes 1, 1 becomes 0.

Left shift  <<

Slides bits to the left by n positions, filling zeros on the right. 1 << 3 moves the bit at position 0 to position 3, giving 0b1000 = 8. Equivalent to multiplying by 2n.

Right shift  >>

Slides bits to the right, dropping the ones that fall off. Equivalent to integer division by 2n. (Signed vs. unsigned right shift matters for negative numbers — more on that in pitfalls.)

5. The essential bit tricks

These are the patterns worth memorizing. They come up constantly: in interview problems, embedded code, graphics, compression, cryptography, game development, and low-level systems work.

Check if bit k is set

const isSet = (n & (1 << k)) !== 0;

Build a mask with a 1 only at position k, then AND with n. If the result is non-zero, that bit was on.

Set bit k

n = n | (1 << k);

Clear bit k

n = n & ~(1 << k);

~(1 << k) is a mask of all 1s except position k. ANDing with it wipes only that bit.

Toggle bit k

n = n ^ (1 << k);

Check even / odd

const isOdd = (n & 1) === 1;

The lowest bit is the "1s" place; if it's on, the number is odd.

Multiply / divide by 2

n << 1   // n * 2
n >> 1   // n / 2  (integer, drops remainder)

Is n a power of 2?

const isPow2 = n > 0 && (n & (n - 1)) === 0;

Powers of 2 have exactly one bit set. Subtracting 1 flips that bit off and turns every lower bit on (e.g. 8 = 0b1000, 7 = 0b0111). ANDing them gives 0. Elegant.

Isolate the lowest set bit

const lowest = n & -n;

Uses two's-complement negation. If n = 0b0110_1100, then -n = 0b1001_0100, and AND gives 0b0000_0100 — only the rightmost 1 survives.

Count set bits (Brian Kernighan's trick)

function popcount(n) {
  let count = 0;
  while (n) {
    n &= n - 1;   // turns off the lowest set bit
    count++;
  }
  return count;
}

Loops exactly once per set bit — much faster than scanning all 32 positions when the number is sparse.

Swap two variables without a temp

a ^= b;
b ^= a;
a ^= b;

Cute but almost never useful in real code — a temp variable is clearer and often faster. Good to recognize if you see it.

6. Bitmasks: an int as a bag of flags

A common real-world use: pack several yes/no flags into one integer instead of using a struct with a bunch of booleans. 

// declare flags as powers of 2
const READ    = 1 << 0;   // 0b0001
const WRITE   = 1 << 1;   // 0b0010
const EXECUTE = 1 << 2;   // 0b0100
const ADMIN   = 1 << 3;   // 0b1000

// combine with OR
let perms = READ | WRITE;         // 0b0011

// test with AND
if (perms & WRITE) { /* has write */ }

// grant execute
perms |= EXECUTE;                  // 0b0111

// revoke write
perms &= ~WRITE;                   // 0b0101

You'll see this in filesystem permissions (chmod!), CSS event listeners, graphics APIs, compression flags — anywhere someone wants a compact, fast "set of options".

7. Common pitfalls

  • Operator precedence. a & b == c parses as a & (b == c) in most C-family languages, which is almost never what you want. Wrap comparisons in parentheses: (a & b) == c.
  • Signed vs. unsigned right shift. In JavaScript, >> preserves the sign bit (arithmetic shift) while >>> fills with zeros (logical shift). C and C++ leave signed right-shift as implementation-defined — check your types.
  • Integer width. Bit tricks assume you know how many bits you have. In JavaScript, bitwise operators secretly convert to 32-bit signed integers, so 1 << 31 is negative. Python integers are arbitrary precision, so no such trap. C uses whatever type you gave it — uint8_t vs. int matters.
  • Don't reach for bit tricks first. They're fast and compact but often less readable. Use them when there's a real reason (hot loop, embedded, protocol parsing) or when the intent is clearly a bitset. Otherwise prefer normal code.

8. Practice exercises

Try these before peeking at the hints. Use the playground to check answers.

  1. Write a function getBit(n, k) that returns the value (0 or 1) of the bit at position k of n.
    hint (n >> k) & 1
  2. Write a function that reverses the 8 bits of a byte. 0b10110000 → 0b00001101.
    hint Loop i from 0 to 7. For each bit that's on in the input, set bit 7 - i in the output.
  3. Count how many bits differ between two numbers (Hamming distance).
    hint popcount(a ^ b). XOR gives 1s exactly where they differ.
  4. Given a non-negative integer n, find the position of its highest set bit. (Return -1 if n is 0.)
    hint Right-shift until n becomes 0, counting shifts. Or in many languages there's a builtin: Math.clz32 in JS, __builtin_clz in C.
  5. Given a set of items encoded as a bitmask, enumerate every subset.
    hint For a mask s, iterate t = s; t; t = (t - 1) & s. Classic competitive-programming trick.
