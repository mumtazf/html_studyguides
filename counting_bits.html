Counting Bits — Range Doubling

LeetCode 338 · O(n) DP with power-of-2 ranges

Core Intuition

1
When b is a power of 2, numbers in [b, 2b) are identical to numbers in [0, b) — just with the b-bit switched on.
2
So the bit count for x + b is simply the bit count for x plus 1.
3
Each round doubles the known range: [0,1) → [0,2) → [0,4) → [0,8) …

Why [b, 2b)?

Annotated Code

class Solution: def countBits(self, n: int) -> List[int]: ans = [0] * (n + 1) # result array, ans[0] = 0 x = 0 # index into [0, b) b = 1 # current power-of-2 range start # [0, b) is already calculated while b <= n: # while range fits # generate [b, 2b) or [b, n) from [0, b) while x < b and x + b <= n: # x in [0,b) and within bounds ans[x + b] = ans[x] + 1 # copy + add the b-bit x += 1 x = 0 # reset for next round b <<= 1 # b = 2b (next power of 2) return ans

Complexity

T
Time: O(n) — Each index is written exactly once. The outer loop runs log(n) times, and the total inner-loop iterations across all rounds sum to n.
S
Space: O(n) — The output array of size n + 1.
